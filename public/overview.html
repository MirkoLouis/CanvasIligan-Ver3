How It Works

Ever wondered how CanvasIligan turns your search for "a small motor that shakes" into a list of specific vibration motors available in local stores? Here’s a step-by-step look at our semantic search process.

  1. 1. You Start the Search
    A user (you!) submits a query through the client-side interface. The browser's JavaScript constructs and dispatches an asynchronous HTTP GET request to the Node.js application server's `/api/search` endpoint, with the query string URL-encoded.
  2. 2. Understanding Your Query
    Our server sends your query to a special machine learning service. This service uses a powerful language model to analyze the text and convert it into a "vector"—a numerical representation of its meaning.
  3. 3. Finding the Closest Match
    This query vector is then compared against a pre-built index of vectors for every product in our database. Using a high-speed library called Faiss, we find the products whose meanings are closest to your query. This is how we match concepts, not just keywords.
  4. 4. Gathering Product Details
    The search returns a ranked list of product IDs. Our main server takes this list and fetches all the important details for each product (like name, price, description, and store availability) from our database.
  5. 5. Displaying the Results
    Finally, all this information is sent back to your browser. The page then dynamically displays the list of relevant products, ready for you to explore.
